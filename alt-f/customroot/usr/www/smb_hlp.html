Samba Setup Help

In this page you can configure Samba, allowing MS-Windows computers to access box shares, and to make available to the box shares defined in other computers.
A share is a network offer of an existing folder.

  • In the upper Host details section you might configure how the box appears in the MS-Windows network browser:

    • Host name the network name used to access the box.
    • Host description the box description.
    • Workgroup the MS-Windows workgroup the box will bellong to.

  • In the Folders to export to other hosts section you can create shares, one per line:

    • Disable fast way to enable or disable a share.
    • Folder local folder name that will be offered as a share.
      If empty, the share (not the folder!) will be removed when the Submit button is hit.
    • Browse opens the Folders Browse window, which allows selecting an existing folder (or creating a new one).
      The selected folder will automaticaly fill-in the previous Folder entry field.
    • Share Name the name of the offered share; if empty the folder name will be used.
    • Comment F text that will appear in the MS-Windows explorer near the share name; if empty the folder name will be used.
    • Allow selects who can access the share.
      Notice that Samba will not override the filesystem folder permissions, i.e., the folder associated with a share also has to have the right permissions (Setup->Folders->Permissions, or just use the Browse button).
      The Users and Public shares and folders, which are automatically created when the first user is created, have the correct permissions for the intended usage, use them as a pattern.
      For further details read the "Folder Ownership and Access Permissions" online help.
      • anybody: everybody can access the share, even anonymous or guest users.
      • nonpublic: reserved for the Users home folder.
        Each user's Home folder can only be accessed by himself when he is authenticated.
      • user: only the specified user will be able to access the share.
      • +group: only members of the specified group will be able to access the share.
    • Browseable the share is browseable.
    • Read Only no writing is allowed.
    • Inherit Perms created files or folders inherit the base folder ownership and permissions.

  • In the Folders to import from other hosts section you can make other computers shares available to the box.
    After an entry is filled-in and the Submit button is hit, a Mount button will appear, which will make the remote share localy available; when mounted, an unMount button will appear instead, which if hitted will unmount (unmap) the remote share.

    • Host the computer name where the share is available.
      If empty, the whole entry will be removed when the Submit button is hit.
    • Share the remote share name.
      If empty, the whole entry will be removed when the Submit button is hit.
    • Discover opens a new window and performs a SMB/CIFS network scan, showing the existing computers and offered shares; clicking a share will fill-in the previous Host and Share entry fields.
    • Local Folder the local folder where the remote share contents will be available.
      If the folder is not empty, its contents will (temporarily) disappear while the remote share is mounted, and will reappear when the remote share is unmounted.
    • Search opens the Folders Browse window, which will fill-in the Local Folder entry field with the selected folder.
    • Mount Options specify the desired mount options; the default values will generally be OK.

  • The Advanced button allows using SWAT, the Samba Web Administration Tool, which opens a new window after asking for the root credentials.
    SWAT allows a Samba administrator to configure the complex smb.conf file, producing a configuration file that will not be fully recognized by the Alt-F webUI; you should thus use either SWAT or the Alt-F Samba webUI to configure Samba.
    If you install the samba-doc Alt-F package you can then use the SWAT online help facilities.
