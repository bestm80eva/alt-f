Status Page Help

This page shows the box status.
If the Autorefresh checkbox at the botton is checked, then each status section will refresh itself between every 10 to 20 seconds. Leaving it checked will avoid the disks to spin down and will consume some CPU power, so some lenghly tasks might tale longer to accomplish. If the disks are already spindown they will not spinup when the page is displayed.

There are several Status sections, not all will be shown if there are no status to display for them:

  • System: shows the box system status.
    • Temperature: is the box internal temperature. It will be affected by ambient temperature and by disks temperature.
    • Fan speed: the current fan speed. The fan speed is automaticaly adjusted according to the box temperature. You can change the relationship bewteen fan speed and temperature in the Services->System->sysctrl->Configure section.
    • Load: The number of processes trying to execute simultaneously on the CPU.
    • CPU: how busy the CPU is
    • Swap: amount of disk space used as memory; when the system needs more memory than physical memory is available, a special disk partition called swap is used ans some less frequently used applications and data are moved to the disk.
  • Network: shows network status
    • Speed: can be 10, 100 or 1000 Mega bits per second, depending on the negotiated rate with the nearby connected device.
    • MTU: also called "jumbo frames"; they can improve network throughtput when transfering large files, but both sides of the transfer must support it.
      To change it, go to Setup->Host. Try to use values lower than 3700 bytes and/or 7500 bytes.
      Changing it right after powerup is fine, but changing it when the system has already beeing active for a some time can actually degrade network performance.
    • Rx/Tx the amount of data received and sent by the box
    • MAC: the network chip identification, which is fixed and world-unique
  • Disks: shows available disks
    • Bay: where the disk is attached: the left, right or USB port
    • Dev: the linux device name used to identify the disk
    • Model: the disk model
    • Capacity: the disk Capacity
    • Power: the disk power status, can be "active or iddle", standby or unknown.
    • Temp: the disk internal temperature
    • Health: the disk health, as reported by SMART, or unknown, if the drive does not support SMART or is in standby.
      If you see a message in red, replace the disk as soon as possible, as the disk is likely to fail in the next hours or days.
      If the message appears in blue it means that some disk parameter has failed in the past.
      You can examine the SMART log in Disk->Utilities->Health->Show Status.
  • RAID: show RAID status
    • Dev: the name used by linux to identify the RAID device
    • Capacity: the RAID capacity
    • Level: the RAID type, can be linear(JOBD), raid0, raid1 or raid5
    • State:
    • Status: the RAID status, can be OK or degraded. If 'degraded' appears in red, the RAID is not providing any redundancy, as one disk has failed and must be replaced. Go to Disk->RAID.
    • Action: can rebuilding or resyncing.
    • Done: shows the percent completion of th the Action operation
    • ETA: shows the expected time to complete the Action operation
  • Mounted Filesystems: shows mounted local filesystems
    • Dev: the name used by linux to identify the filesystem; except for RAID, it has the same name as the disk where it resides, plus the number of the disk partition.
    • Label: a filesystem can have an human-readable label, such a Pictures, or Videos.
    • Capacity: the filesystem capacity
    • Available: the filesystem capacity still available for use, and a bar showing the percentage already used. li>FS: the kind of filesystem, can be ext2/3/4, fat or NTFS
    • Mode: can be RO (Read-only) or RW (Read and Write). A filesystem is RO if when it was checked at powerup or at plug-time some errors couldn't be fixed -- to fix them a manual operation is needed.
    • Dirty: the filesystem is in an inconsistent state, you should go to Disk->Filesystem->FS Operation and selecte "Check". "ext2" filesystems are marked as dirty as soon as they are mounted.
    • Automatic FSCK in: the number of mounts or days left until an automatic filesystem check will be done at powerup or mount time.
      To avoid an inoportune lenghly check, that will certainly happen when you most need your files, you should manualy start a filesystem check using Disk->Filesystems->Operation->Clean
  • Mounted Remote Filesystems: shows filesystems from remote hosts localy available
    • Host: the host where the filesystem resides
    • Remote Dir/Share: the directory or share of the remoter host
    • Local Dir: the local directory where the filesystem is available
    • Capacity: the filesystem capacity
    • Available; the amount of free storage on the filesystem
    • FS: the filesystem type, can be NFS otr cifs/smb
  • Remotely Mounted Filesystems: shows local filesystems in use by remote hosts
    • Host: the host that is using the local filesystem
    • Dir/Share: the local directory or share name being used
    • FS: the kind of filesystem, can be NFS or cifs (smb)
  • Backup: shows currently running backup
    • ID; the backup id, set when configuring backups
    • Directory: the directory being backup
    • State: the backup status, can be "In Progress" or Queued; only one backup is performed at a time, if others are schedule for the same time they will wait for the current one to finish
  • Filesystem Maintenance: shows currently filesystem maintenance operations
    • Dev: The filesystem linux identifier
    • Label: The filesystem label
    • Operation: the maintenence operation occuring, can be checking, formating, converting, shrinking, enlarging or wiping.
  • Printers: shows available printers
    • Name: the printer identification
    • Model: the printer model
    • Jobs: the number of print jobs awaiting to be printed
