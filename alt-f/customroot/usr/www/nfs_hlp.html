NFS Setup Help

In this page you can configure NFS, allowing UNIX computers to access this box shares, and to make available to the box existing shares in other UNIX computers.
A share is a network offer of an existing folder.

  • Folders to export to other hosts

    Specifies folders that can be NFS mounted by other computers. Only NFS-v3 is supported.
    • Disable: Check to temporary stop exporting the share.
    • Folder: The folder to export. Note: To export the same folder to different hosts with different options, enter the same folder on different lines for each host.
    • Search: Hit to browse the local folder structure. Hitting "OK" will fill-in the Folder field.
    • Allowed hosts: the hosts allowed to mount the share. Can be:
      • anonymous: a single * character will match all clients.
      • single host: a host name setup under Setup->Hosts, or an IP address
      • IP networks: a (sub) network specified by address/netmask
      • wildcards: wildcard characters * and ?, or class lists within [square brackets].
      • netgroups: NIS netgroups may be given as @group (not supported)
    • Export Options: Any NFS allowed option, comma separated, no spaces.
    • Options: Opens a new window with a list of the most frequently used options and the currently selected ones; hit CTRL while clicking to select multiple options. Options of the form option=value does not preserve the current value.

  • Folders to import from other hosts

    Specify remote folders to be locally mounted, e.g. to make them available under other protocol.
    • Disable: Check to temporary stop mounting the share..
    • Host: The remote hostname (or IP address) that exports the share.
    • Folder: The remote folder to mount locally.
    • Discover: Hit to NFS browse the network; hitting a line will fill the Host and Folder fields.
    • Local Folder: Local folder where the remote share should be made available. It is recommended to use an empty folder directly under "/mnt"; if necessary create the folder, e.g. "/mnt/remoteNFS".
    • Search: Hit to browse the local folder structure (and possibly create the mount folder).
    • Mount Options: NFS mount options, comma separated, no spaces.
    • Options: Opens a new window with a list of the most frequently used mount options and the currently selected ones; hit CTRL while clicking to select multiple options. Options of the form option=value does not preserve the current value. After an entry is filled-in and the Submit button is hit, a Mount button will appear, which will make the remote share locally available; when mounted, an unMount button will appear instead, which if selected will unmount (unmap) the remote share.

  • Delay NFS start on boot:

    NFS stores some client information on disk, for recovery on reboot in case of server or client crash. That information is stored in a sub-directory where Alt-F packages are stored. At boot time, when NFS starts, the Alt-F directory might not yet be available and such recovery information is missing.
    Checking this checkbox delays the NFS start until the Alt-F directory becomes available. Notice that if the checkbox is checked and the Alt-F directory never appears, then NFS will not auto-start.
